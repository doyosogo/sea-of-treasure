# Security Policy

## Reporting a Vulnerability

If you discover a security issue, do not open a public issue. Report it privately to the repository maintainers and include:

- a short description of the vulnerability
- the affected area of the project
- steps to reproduce it
- any proof of concept or logs that help confirm the issue

## Supported Versions

Security fixes are provided for the current main branch and the latest released version.

## Responsible Disclosure

Please allow time for the issue to be reviewed and addressed before publishing details publicly. The goal is to coordinate a fix before disclosure.
